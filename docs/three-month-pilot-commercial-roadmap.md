# Three month pilot and commercial roadmap

This app should be built and tested as a sellable site management product, not only as a personal tool.

## Product aim

SiteProg should prove that a site manager can control programme, inspections, daily walks, trade actions and close-out from one mobile-first system.

Core product sentence:

The programme shows what should be happening. The 8am walk checks what is happening. Stage inspections record quality. Trade actions get issues closed out.

## Three month pilot objectives

### Month 1: Prove the workflow

- Add live plots.
- Generate a master programme.
- Generate a 2-week programme.
- Complete stage inspections.
- Record failed checks.
- Create trade actions.
- Use the 8am walk daily.
- Record DABS as a standalone PM meeting record.

Success evidence:

- Number of plots managed.
- Number of inspections completed.
- Number of defects/actions created.
- Number of actions closed.
- Number of trades using the issued lookahead.

### Month 2: Prove trade control

- Issue trade 2-week lookahead in spreadsheet format.
- Track open actions by trade.
- Track sent to trade status.
- Track fixed awaiting verification.
- Track rejected close-outs.
- Improve inspection templates from real site use.

Success evidence:

- Reduction in missed inspections.
- Reduction in untracked trade issues.
- Faster action close-out.
- Better visibility of which trades are blocking progress.

### Month 3: Prove commercial value

- Create a clean demo dataset.
- Create screenshots or screen recordings.
- Export example trade lookahead.
- Export example inspection record.
- Export example action log.
- Prepare a simple pitch deck.
- Prepare pricing options.

Success evidence:

- Daily active use.
- Repeatable site workflow.
- Clear before and after benefit.
- Evidence that trades understand the outputs.
- Evidence that the app saves time or reduces missed items.

## Minimum sellable version

The product should have:

1. Master programme.
2. 2-week programme.
3. Trade spreadsheet lookahead.
4. Stage inspection checklists.
5. Defect and action log.
6. 8am site walk.
7. DABS PM meeting record.
8. Photo evidence.
9. Export or copy functionality.
10. Simple dashboard.

## Buyer story

Potential buyers should understand the app in one minute:

- Site managers are overloaded.
- Programme, quality and trade actions are often spread across diaries, WhatsApp, Excel and memory.
- SiteProg brings the daily control loop into one place.
- It creates evidence of inspections, trade actions and programme risks.
- It gives trades a simple spreadsheet-style lookahead.

## Commercial options to test

- Per site monthly licence.
- Per company monthly licence.
- Per user monthly licence.
- One-off internal licence for regional housebuilders.
- White-label version for developers or contractors.

## Data and trust requirements

Before selling, the app will need:

- Clear login and user roles.
- Company/site separation.
- Backup and sync.
- Basic privacy policy.
- Exportable data.
- Removal of personal demo names.
- Professional app name and brand decision.

## Build priorities before trying to sell

1. Camera/photo upload.
2. Export trade lookahead to CSV or PDF.
3. Export inspection records.
4. Export action log by trade.
5. Add proper user login.
6. Add cloud database.
7. Improve dashboard metrics.
8. Add onboarding for new sites.
9. Add pilot metrics.
10. Create a demo mode.
